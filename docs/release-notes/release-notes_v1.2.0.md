# リリースノート — v1.2.0

**リリース日:** 2026-02-02

## 概要
記号指定機能を拡張し、「チェックボックス方式」だけでなく「テキスト入力方式」による記号指定が可能になりました。  
さらに、入力された記号を「含む／除外する」で切り替えられるようにしました。

---

## 追加・変更点

### 1. 記号の指定方法を追加（チェックボックス／テキスト入力）
- 記号指定に **「指定方法」**（チェックボックス／テキスト入力）を追加しました。
- 初期状態は **チェックボックス方式** を選択します。
- **テキスト入力方式** を選択すると、記号チェックボックス関連UIが非表示になり、テキスト入力欄が表示されます。

### 2. 「含む／除外する」切替を追加（テキスト入力方式）
- テキスト入力欄で指定した記号について、以下を切り替え可能にしました。
  - **含む**：入力した記号のみを使用対象にする
  - **除外する**：入力した記号を使用対象から除外する

### 3. 「記号なし」選択時の表示制御を改善
- **「記号なし」** を選択した場合、以下の記号関連UIがまとめて非表示になります。
  - 「指定方法（チェックボックス／テキスト入力）」
  - 説明文
  - 記号テキスト入力欄（含む／除外する含む）
  - 記号チェックボックス一覧
  - 「記号を表示」ボタン、「全選択／全解除」ボタン（JSで挿入されるUI含む）

### 4. 記号入力パターンの解釈強化（テキスト入力方式）
- 次のような入力表現を解釈し、記号集合として採用できるようにしました。
  - 空白（半角／全角）区切り
  - カンマ区切り
  - 読点区切り
  - クォート（ダブル／シングル）での囲み
  - 括弧（[] {} ｛｝ 「」 『』 【】など）での囲み
  - **区切りなし（例：`@-:'"[{,]}`）でも入力された文字を記号として扱う**（※一部例外あり）

> ※カンマ（`,`）は区切りとしても使われるため曖昧になりやすく、特定の表記では区切りとして扱われます。
> カンマ自体を記号として明示したい場合は `,,` や `","` / `[,]` / `{,}` などの表記を推奨します。

### 5. `section-title` と `small` 併用時の色の上書き対策
- `section-title` と `small` を併用した見出し（例：「指定方法」）が `small` 側の色指定で青字にならない問題に対応しました。
- 対策として以下のCSSを追加することで、青字（accent）を優先できます。

```css
.section-title.small {
  color: var(--accent);
}
```
